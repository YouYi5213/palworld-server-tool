# 部署清单 — 给另一个会话

## 1. 前置环境

```bash
# 确保 Python 3.11+ 可用
python3 --version

# 编译 Go 需要 Go 1.21+
go version
```

## 2. module/ 目录（存档解析，未推送 GitHub，需手动同步）

把 `H:\Hermes Project\palworld-server-tool\module\` 整个目录复制到服务器项目根目录，结构如下：

```
palworld-server-tool/
├── module/
│   ├── sav_cli.py          # Python 存档解析脚本
│   ├── requirements.txt    # 依赖说明
│   ├── README.md           # 部署说明
│   └── palsav/
│       └── palsav/         # 内置 palsav 库（兜底版）
│           ├── archive.py / gvas.py / paltypes.py / json_tools.py / palsav.py
│           └── rawdata/
```

## 3. 安装 PalWorld 1.0 兼容的 palsav 库

```bash
# 先装 pip 版（兜底）
pip install palworld-save-tools

# 再拉 1.0 兼容的 fork 覆盖内置库
git clone https://github.com/deafdudecomputers/PalworldSaveTools.git
cp -r PalworldSaveTools/src/palsav/palsav/* module/palsav/palsav/
```

如果 fork 拉不下来，先不覆盖也行，但 1.0 存档可能解析不全。

## 4. 构建前端

```bash
git submodule update --init --recursive
cd web && pnpm install && pnpm build && cd ..
cd pal-conf && npx vite build && cd ..
cp pal-conf/dist/index.html pal-conf.html
cp pal-conf/dist/assets/* assets/
```

## 5. 编译 Go

```bash
go build -o pst main.go
```

## 6. 运行并验证

```bash
./pst -config config.yaml
```

启动后观察日志，存档同步时会打印：
```
Parser: palsav (deafdudecomputers)    ← 确认用的 1.0 兼容库
File: /tmp/.../Level.sav
Players: 3, with pals: XX, Guilds: 1   ← XX > 0 表示帕鲁解析成功
PUT /api/player → 200
Done
```

## 7. 如果帕鲁解析失败

运行时加 `--debug` 参数手动测试，会导出完整 GVAS JSON 到存档目录：

```bash
python3 module/sav_cli.py -f /path/to/Level.sav --request http://127.0.0.1:18080/api/ --token <JWT_TOKEN> --debug
```

这会生成 `Level.sav.json`，发给我分析数据结构。
